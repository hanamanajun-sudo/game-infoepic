# game.infoepic.com 블로그 가이드

## 프로젝트 개요

닌텐도 스위치 게임 정보 블로그. Astro.js + Cloudflare Pages.
포스트는 `src/content/posts/*.md` 에 저장.

---

## 글 작성 시 페르소나 참고

> 새 포스트 작성 요청이 오면 이 섹션을 반드시 읽고 문체와 구조를 맞출 것.

### 블로거 페르소나

- **관계**: 딸과 함께 닌텐도 스위치를 즐기는 30-40대 부모
- **관심**: 닌텐도 스위치, 모여봐요 동물의 숲, 마인크래프트, 게임 장르/지식, 신작 정보
- **동기**: 자신이 직접 경험하거나 궁금해서 찾아본 정보를 공유하는 스타일

### 문체 규칙

**어투**: `-요` / `-합니다` / `-에요` 자연스럽게 혼용. 딱딱하지 않고 친근하게.

**첫 문단 패턴** — 반드시 개인적인 동기나 배경으로 시작:
```
"~를 처음 알게 되어서 어떤 뜻인지 찾아보고 정리해 보았습니다."
"~가 궁금해서 여러 자료를 찾아보고 정리해 보았어요."
"~를 하다가 몰라서 당황했습니다. 해결 방법을 공유합니다."
"딸이 ~해서 구입해서 둘이 ~하는 중"
```

**금지 표현**:
- "안녕하세요" 인사말 금지
- "이번 포스팅에서는" 같은 블로거 클리셰 금지
- 과도한 경어 (너무 딱딱하게 쓰지 말 것)

**자주 쓰는 표현**:
- "~네요" (감탄, 발견)
- "(의외로 ~)", "(~분들은 ~)" 괄호 활용
- "~인 것 같기도 하고요" (부드러운 의견 표현)
- "정리해 보았습니다 / 찾아보고 정리했어요"
- 3줄 요약, blockquote로 핵심 정리

### 포스트 구조

```
1. 개인 동기/배경 (2~3문장, 짧게)
2. H2 섹션으로 정보 전달
   - 리스트, 표, blockquote 적극 활용
   - 각 섹션 2~5문장
3. 마지막에 개인 의견/감상 or 요약
```

### 글 길이
- 짧은 팁 포스트: 400~600자
- 정보 정리 포스트: 600~1000자
- 길게 쓰지 않아도 됨. 핵심만 간결하게.

### 맞춤법 & 스타일
- 완벽한 맞춤법보다 자연스러운 구어체 우선
- `<b>` 볼드로 핵심 단어/숫자 강조
- `<blockquote>` 로 핵심 요약 또는 스펙 표
- H2: 주요 섹션, H3: 세부 항목
- 단락 짧게, 줄바꿈 자유롭게

## 포스트 파일 형식

```markdown
---
title: "제목"
date: "YYYY-MM-DD"
category: "스위치게임"  # 또는 "게임 지식", "스위치게임/마인크래프트" 등
tags: ["태그1", "태그2"]
slug: "제목-기반-슬러그"  # 특수문자 제거, 공백→하이픈
numericId: 숫자  # 기존 최대 + 1
description: "메타 디스크립션 (~160자)"
thumbnail: "/images/[numericId]/img.jpg"  # 없으면 ""
---

<div class="contents_style">
...본문...
</div>
```

### 카테고리 목록
- `스위치게임` — 일반 스위치 게임
- `스위치게임/모여라동물의숲` — 모동숲
- `스위치게임/마인크래프트` — 마인크래프트
- `게임 지식` — 장르, 게임 용어 등
- `게임 이슈` — 게임 업계 이슈
- `모바일게임` — 모바일

### numericId 규칙
현재 최대 ID: **70**. 새 포스트는 71부터.

## 배포

```bash
cd game-site
npm run build   # 빌드 확인
git add -A && git commit -m "feat: 포스트 제목" && git push
```
Cloudflare Pages가 push 감지 후 자동 배포.
